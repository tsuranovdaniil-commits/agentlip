# Правила работы с Git (Командная разработка)

1. **НИКОГДА не делай `git push` автоматически.**
2. Пуш в репозиторий разрешен только после явной команды пользователя (например: "пушить", "можешь пушить").
3. Перед любым коммитом и пушем **ОБЯЗАТЕЛЬНО** выполняй `git pull` (слияние удаленных изменений), так как над проектом работает несколько человек.
